## 6. Design System & Visual Language

### Overall Design Philosophy

```
Premium Minimalism + Futuristic Luxury + Enterprise Grade
```

Inspired by: **Apple × Linear × Stripe × Vercel × Notion**

The design language prioritizes clean, spacious, premium aesthetics with an enterprise feel. Every visual decision should reinforce trust, intelligence, and sophistication.

### Core Design Principles

1. **Simplicity Over Complexity** — Every element must earn its place
2. **Spacious Layouts** — Generous whitespace, breathing room between sections
3. **Strong Typography** — Typography carries the hierarchy, not color
4. **Purposeful Animation** — Smooth, subtle micro-interactions only
5. **Consistency** — Unified visual language across all touchpoints

### UI Specifications

| Property | Value |
|---|---|
| **Border Radius** | `12px – 20px` (soft, refined corners) |
| **Shadows** | Soft, layered, natural (never heavy or harsh) |
| **Borders** | Thin `1px` neutral gray/platinum borders |
| **Animations** | Smooth, subtle, purposeful (premium micro-interactions) |
| **Effects** | Frosted glass, soft blur, metallic reflections, minimal gradients |
| **Icons** | Minimal outline icons with consistent stroke width |
| **Components** | Spacious cards, premium buttons, elegant forms, refined dashboards |

### What to Avoid

- ❌ Bright / neon colors
- ❌ Visual clutter
- ❌ Excessive gradients
- ❌ Neon effects or glow
- ❌ Cartoon / playful styles
- ❌ Flashy or distracting animations
- ❌ Heavy skeuomorphism
- ❌ Generic stock imagery

## 7. Typography

### Font Stack

| Usage | Font | Weight(s) | CSS Variable |
|---|---|---|---|
| **Primary Sans-Serif** | Geist | System default | `--font-geist-sans` |
| **Monospace** | Geist Mono | System default | `--font-geist-mono` |
| **Body / UI** | Inter | 400, 500, 600, 700 | `--font-inter` |
| **Display / Headings** | Syne | 400, 500, 600, 700 | `--font-syne` |

### Typography Rules

- **Headings** use `Syne` for a distinctive, modern display feel
- **Body text** uses `Inter` for optimal readability
- **Code / technical content** uses `Geist Mono`
- Apply `antialiased` rendering globally
- Letter-spacing: tight for large headings (`-0.05em`), normal for body
- Line-height: generous for readability (`1.5` – `1.75` for body)

### Font Loading

All fonts are loaded via `next/font/google` with:
- Subset: `latin`
- Display: `swap` (automatic via Next.js)
- Variable font declarations for CSS custom property usage

### Favicon Implementation

The site implements theme-aware favicons using the `prefers-color-scheme` media query:

```html
<!-- Light mode -->
<link rel="icon" href="/icon-light-32x32.png" media="(prefers-color-scheme: light)" />
<!-- Dark mode -->
<link rel="icon" href="/icon-dark-32x32.png" media="(prefers-color-scheme: dark)" />
<!-- Universal SVG fallback -->
<link rel="icon" href="/icon.svg" type="image/svg+xml" />
<!-- Apple Touch -->
<link rel="apple-touch-icon" href="/apple-icon.png" />
```

### Logo Usage Rules

- Maintain minimum clear space around the logo equal to the height of the icon
- Do not distort, rotate, or apply effects to the logo
- Do not place the logo on busy or low-contrast backgrounds
- Do not recreate or modify the logo without authorization
